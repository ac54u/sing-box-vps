### 💾 第二阶段：配置备份功能 ✅
**目标**：完整的配置管理系统，支持备份、恢复、历史查看

#### 新增功能函数（229 行代码）

##### 1. `backup_init()` - 初始化备份系统
- 创建备份目录：`/etc/s-box/backups/`
- 初始化备份日志：`/var/log/sing-box-vps/backup.log`

##### 2. `backup_config()` - 备份当前配置
- 收集备份范围：
  - JSON 配置文件：`sb.json`, `sb10.json`, `sb11.json`
  - 证书与密钥：`cert.pem`, `private.key`, ACME 证书等
  - 元数据记录：系统信息、Sing-box 版本、时间戳
- 生成备份包：`tar.gz` 格式
- 计算校验和：SHA256 校验码用于完整性验证
- 记录备份索引：`index.log` 文件

##### 3. `view_backups()` - 查看备份历史
- 列表格式显示所有备份
- 显示备份时间、版本、大小、状态
- 为用户选择恢复提供参考

##### 4. `restore_backup()` - 恢复指定备份
- **安全机制**：
  - 恢复前自动备份当前配置为 `auto_backup_<timestamp>`
  - 校验包完整性（SHA256 验证）
  - 验证 JSON 配置有效性
- **恢复流程**：
  1. 停止 sing-box 服务
  2. 恢复配置文件和证书
  3. 启动 sing-box 服务
  4. 验证服务状态（10 秒检查）
  5. 失败自动回滚到 `auto_backup`
- 支持 Alpine 和标准 Linux 系统

##### 5. `delete_backup()` - 删除备份
- 安全确认机制
- 更新备份索引日志
- 删除对应的备份文件

##### 6. `backup_menu()` - 备份菜单驱动
- 友好的菜单界面
- 子菜单递归调用
- 操作完成后自动返回菜单

### 🔍 技术实现细节

#### 性能优化技术方案

**并行 curl 调用示例**：
```bash
v4v6(){
{
v4=$(curl -s4m5 icanhazip.com -k)
} &
{
v6=$(curl -s6m5 icanhazip.com -k)
} &
wait
}
```
- 利用 bash 后台进程（`&`）并行执行网络请求
- 使用 `wait` 同步所有后台任务完成
- 总时间 = max(curl_time)，而非 sum(curl_time)

#### 备份恢复安全措施

1. **前置自动备份**：恢复前自动保存当前配置
2. **完整性校验**：SHA256 哈希验证备份包
3. **配置验证**：JSON 语法检查
4. **服务验证**：启动后 10 秒内检查服务状态
5. **自动回滚**：服务启动失败时自动恢复

#### 日志记录

- 所有备份操作记录到：`/var/log/sing-box-vps/backup.log`
- 时间戳格式：`[YYYY-MM-DD HH:MM:SS]`
- 包含操作成功/失败的详细信息
- 便于故障排查和审计
